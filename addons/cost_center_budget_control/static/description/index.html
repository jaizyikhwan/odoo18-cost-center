Cost Center & Budget Control

Odoo 18 CE LGPL-3 OCA-style v18.0.1.0.0

Enterprise-grade cost center hierarchy management, workflow-driven budget planning, real-time budget utilization tracking, and proactive threshold control — built specifically for Odoo 18 Community Edition following OCA structural conventions.

Why this module?

Standard Odoo analytic accounting excels at cost tracking but lacks native mechanisms to enforce department-level budget limits before overspending occurs. Managers are forced to rely on retroactive reports that only highlight overruns after transactions are posted.

This module fills that gap by validating transactions in real time during the posting workflow — checking projected expenditures against approved departmental budgets before finalizing entries.

Key Features

  • Hierarchical cost centers (parent-child)
  • Workflow-driven budget plans (state machine)
  • Real-time actual_amount aggregation (JSONB SQL)
  • Programmatic overhead allocation engine
  • Balanced journal entry generation
  • Idempotent allocation references
  • Configurable threshold validation (70/90/100%)
  • Role-based budget override controls
  • Multi-company isolation (record rules)
  • Analytic distribution integration
  • Database indexes (incl. GIN on JSONB)
  • Pivot / graph reporting views

Architecture Pillars

1. Workflow Discipline

Budget plans follow a strict state machine: Draft → Submitted → Approved → Closed / Cancelled. Approved plans are locked against modifications to protect historical records.

2. Ledger Accuracy

Programmatic journal entries are perfectly balanced. Rounding residuals are absorbed by the final target line. Idempotency is enforced at the database level via deterministic reference keys.

3. Multi-Company Safety

Every model declares _check_company_auto = True at the module level. Cross-company accounting references are blocked at the ORM layer.

4. Performance-Aware Aggregation

actual_amount aggregation uses parameterized JSONB queries against account_move_line.analytic_distribution with a custom GIN index installed via post_init_hook.

Technical Stack

  • Odoo 18.0 Community Edition
  • PostgreSQL 16 (JSONB queries + GIN indexing)
  • Python 3.11+
  • OCA-style module structure

Installation

git clone https://github.com/jaizyikhwan/odoo18-cost-center.git
cd odoo18-cost-center
docker compose up -d

After Odoo initializes, install Cost Center & Budget Control from the Apps dashboard. The mounted addons/ volume allows real-time local file changes to be reflected immediately on upgrade.
