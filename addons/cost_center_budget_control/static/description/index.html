Cost Center & Budget Control

Odoo 18 CE LGPL-3 OCA-style v18.0.2.0.0

Enterprise-grade cost center hierarchy management, workflow-driven budget planning, real-time budget utilization tracking, PO committed amount monitoring, and immutable budget revision chains — built specifically for Odoo 18 Community Edition following OCA structural conventions.

Why this module?

Odoo 18 Community Edition ships with native account.budget for visibility: aggregated Achieved amount, threshold management in reports, and Committed column. That's good for reporting but stops at the accounting layer.

This module adds the enforcement layer Odoo 18 CE doesn't have: hard posting-block on overrun, role-based override governance, hierarchical cost center tree, programmatic overhead allocation, PO committed tracking with opt-in hard block, and a proper budget revision workflow with immutable version chains.

Key Features

  • Hierarchical cost centers (parent-child)
  • Workflow-driven budget plans (state machine)
  • Real-time actual_amount aggregation (JSONB SQL)
  • PO Committed Amount tracking (opt-in hard block)
  • Budget Revision with immutable chain

  • Role-based budget override controls (group, not context)

  • Mail template over-budget notifications

Architecture Pillars

1. Workflow Discipline

Budget plans follow a strict state machine: Draft → Submitted → Approved → Revised / Closed / Cancelled. Approved plans are locked against modifications to protect historical records. Revised plans are immutable; the active budget is the latest revision.

2. Hard Enforcement with Role-Based Override

When the budget control mode is blocking, posting a journal entry that would exceed the blocking threshold raises UserError before the move is finalized. Override is granted by membership in the group_budget_override_manager security group — not by a context flag that could be set by client code. Every override is logged in the document chatter.

3. PO Committed Amount Integration

Confirmed-but-unbilled Purchase Order lines are aggregated into budget lines as po_committed_amount. The total committed_amount = actual_amount + po_committed_amount mirrors Odoo 18 Enterprise's Committed column. An opt-in setting (Block Purchase Orders) can extend the same hard-block enforcement to RFQ confirmation.

4. Budget Revision Chain

Approved budgets can be revised mid-period. The original is marked revised (immutable history); a fresh editable clone is created, linked via parent_revision_id, and renamed with a (Rev N) suffix. The chain is fully auditable.

    [*] --> Draft
    Draft --> Submitted: Submit
    Submitted --> Draft: Reject
    Submitted --> Approved: Approve
    Approved --> Revised: Revise (clone + original locked)
    Revised --> [*]
    Approved --> Closed: Close
    Submitted --> Cancelled: Cancel
    Approved --> Cancelled: Cancel
    Closed --> [*]
    Cancelled --> [*]

5. Ledger Accuracy

Programmatic journal entries are perfectly balanced. Rounding residuals are absorbed by the final target line. Idempotency is enforced at the database level via deterministic reference keys.

6. Multi-Company Safety

Every model declares _check_company_auto = True at the module level. Cross-company accounting references are blocked at the ORM layer.

7. Performance-Aware Aggregation

actual_amount and po_committed_amount aggregation uses parameterized JSONB queries against account_move_line.analytic_distribution and purchase_order_line.analytic_distribution with a custom GIN index installed via post_init_hook.

Installation

git clone https://github.com/jaizyikhwan/odoo18-cost-center.git
cd odoo18-cost-center
docker compose up -d

After Odoo initializes, install Cost Center & Budget Control from the Apps dashboard. The mounted addons/ volume allows real-time local file changes to be reflected immediately on upgrade.
